# ¿Qué hace esto? Apaga el contenedor de forma segura y destruye la red virtual interna que creó Docker. Como no le pusimos volúmenes locales al archivo, el bucket que creamos (mi-bucket-frontend) y los archivos de React que subiste se borrarán. Esto es ideal porque te deja el entorno 100% limpio para cuando quieras volver a probar el despliegue desde cero.

Para que te quede la total tranquilidad técnica: el AWS CLI no levanta ningún servicio, "daemon" o proceso de fondo (background process) en tu Mac. Funciona bajo el principio de petición-respuesta efímera:

    Tú escribes aws s3 ls y presionas Enter.

    El sistema operativo despierta el binario del AWS CLI.

    El CLI envía la petición HTTP al puerto de tu Floci, recibe la respuesta, te la pinta en la pantalla y se muere inmediatamente.


    Mientras no estés ejecutando un comando manualmente, el AWS CLI ocupa exactamente 0% de CPU y 0 MB de RAM. Es solo código guardado en tu disco duro esperando a ser llamado.

Los únicos componentes que sí consumían recursos de manera constante eran el contenedor de Docker (Floci) y el túnel de Ngrok, los cuales ya apagaste con éxito mediante docker compose down y cerrando la terminal. Tu Mac está ahora mismo completamente libre de procesos de este ejercicio.



aws s3 ls

aws: [ERROR]: An error occurred (InvalidAccessKeyId) when calling the ListBuckets operation: The AWS Access Key Id you provided does not exist in our records.



No, no es normal en el contexto de las pruebas que veníamos haciendo, pero el error tiene una explicación técnica muy lógica.

Te explico exactamente por qué te acaba de salir este error:
1. ¿Por qué falló? (Falta el --endpoint-url)

Ejecutaste el comando así: aws s3 ls.

Al no ponerle el parámetro --endpoint-url=http://localhost:4566, el AWS CLI asume por defecto que estás intentando conectarte a los servidores reales y públicos de Amazon Web Services en internet.

Como en los pasos anteriores configuramos credenciales ficticias (test / test), Amazon revisó sus servidores reales, no encontró ninguna cuenta con la llave test, y te rechazó el acceso con el error: "(InvalidAccessKeyId) ... does not exist in our records".
2. ¿Cómo debe ser el comando?

Recuerda que para hablarle a tu servidor local (Floci), siempre debes indicarle explícitamente a dónde apuntar:


aws --endpoint-url=http://localhost:4566 s3 ls

aws: [ERROR]: Could not connect to the endpoint URL: "http://localhost:4566/"
